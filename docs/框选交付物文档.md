# 交付物文档（结合 Layout-RAG 智能布局系统实现）

## 1. 产品规划设计文档

### 1.1 需求背景与痛点分析
> **具体要求：** 需包含数据支撑，量化说明问题严重程度

*在低压开关柜（Lowvoltage Cabinet）和配电箱的电气设计过程中，面板元件布局是一项核心但极度耗时的工程。*
- **痛点1：高度依赖人工经验，设计周期长。** 当前工程师手动排布一面包含 50+ 元器件（如断路器、接触器、微型开关等）的开关柜面板，往往需兼顾电气尺寸干涉、发热要求，通常耗时 2~4 小时。
- **痛点2：复用率极低且缺乏对账机制。** 历史设计方案未能数字化沉淀。当新方案与旧方案具有 80% 相似度时，工程师依然需要从头查阅图纸，无法一键套用过往成熟方案（旧有方式缺乏基于结构特征的检索能力）。
- **痛点3：容易出错。** 不同的主回路电流（如 `panel_main_circuit_current`）、极数、面板空间限制（宽度/高度维度）的互相约束极其繁杂，手动排导致空间填充率低且易发生干涉。

> **业务目标：** 引入 Layout-RAG（基于大模型与特征检索增强的布局推荐）与 AI Agent 对话系统，将单面柜体的电气排版与选型时间从数小时缩短至 **3-5 分钟** 内，操作人员只需导入参数或语言对话即可获取图纸，整体设计效率预计提升 90% 以上。

### 1.4 AI 应用场景说明
> **具体要求：** AI 赋能点清单、技术选型预研及 AI 效果验收标准

- **AI 赋能点1：基于自然语言及视觉指令的构建（Configurator Agent）**
  - **应用说明：** 使用 LangChain 封装智能体，底层兼容具有 Vision 功能的顶级大模型。用户无需手动逐一输入型号及长宽数据，只需输入“新建一面左出线柜包含5个接触器”或直接抛入草图，AI即可通过声明的 `[add_cabinet, add_panel, add_part]` 工具链构建出结构树。
- **AI 赋能点2：Layout-RAG 基于结构检索增强的设计推荐**
  - **应用说明：** 常规的 RAG 用于文本。本系统创新地运用业务抽象特征，将当前面板转化为向量数组（诸如元件总面积 `total_parts_area`、空间填充率、大型元件比例 `large_part_ratio` 等预设权重，参见 `schema_def`），将其实时送入本地建立的历史打样图纸库计算欧氏距离，向用户推荐出物理重合度最高的过往成果。
- **效果验收标准：** 
  - **Tool Call 精准度：** 自然语言指令转化为 JSON 数据结构时，型号、树结构的准确率 > 95% 且需包含容错回滚机制。
  - **RAG 相关性：** Top-3 推荐模板返回的分数（Score 公式经过 missing/extra 个数衰减处理后），在测试数据集中的预期采纳率应突破 70%。


### 2.2 关键技术选型
> **具体要求：** 核心技术栈及第三方依赖说明

- **前端展现层依赖：**
  - **核心框架：** 采用 **Vue.js 3** 配合全局双向绑定实现页面局部无刷新的实时响应体验，有效控制侧边列表重绘。
  - **UI 与样式：** 采用 **Tailwind CSS**，轻量化原子类极大缩短了内联开发的编写时长并确保深浅度风格统一。
  - **内容渲染：** 强依赖 **Marked.js** 及 **DOMPurify** 解析流式的模型输出，规避 XSS 等越权漏洞，并用于呈现高阶的 Markdown（代码块及 Mermaid 等）。
- **后端通讯与网关层：**
  - **Web 框架：** **FastAPI** + **Uvicorn**，利用其原生 Async 支持和 `StreamingResponse` 提供稳定、高吞吐的 SSE (Server-Sent Events) 输出，与大模型的打字机流式（thinking token / action token）特性结合天衣无缝。
- **AI Agent与 RAG 算法核心（`layout_rag` 组件）：**
  - **智能体框架：** 基于 **LangChain / LangGraph** 实现 ReAct 工具调用能力。动态识别 `current_scheme` 并向 LLM 注册相关的 `human_tools` 处理指令。
  - **特征打分与存储：** 鉴于业务特征不需要数十亿级别的切片数据规模，自研基于纯内存计算与持久化的 `VectorStore.json`。引入轻量级加权欧氏距离和 `diff_info` （extra/missing 差异）衰减函数，替代了笨重的 Milvus 集群，极大降低部署门槛。
  - **领域驱动设计（DDD）：** 基于 `BusinessDomain` 抽象业务基类，通过策略模式对 `LowvoltageCabinet` （低压开关柜）及其 Schema 特征实现进行解耦，满足未来变电箱、集装箱电源等领域的插拔式拓展。
