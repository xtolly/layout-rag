# Layout-RAG 智能辅助设计平台：产品交互流程图

本文档面向业务和产品团队，展示电气工程师在使用本平台时经历的主业务动线与交互链路。去除了底层复杂的技术与接口时序，主要聚焦于**用户动作（User Action）**与**系统反馈（System Feedback）**的体验闭环。

## 1. 业务全景操作主干线

展示了用户从新建项目到最终完成图纸输出的全生命周期主线流程。我们将整个长链路业务操作划分为四个极为清晰的体验段落：**项目起步 -> 对话式配置 -> AI 接管排版 -> 人工柔性定稿**。

```mermaid
flowchart TD
    %% 颜色与样式定义
    classDef userAction fill:#e1f5fe,stroke:#0288d1,stroke-width:2px;
    classDef aiAction fill:#f3e5f5,stroke:#8e24aa,stroke-width:2px;
    classDef sysAction fill:#fff3e0,stroke:#f57c00,stroke-width:2px;

    subgraph Phase1 [① 项目初始化与参数设定]
        A(创建高/低压电气配置柜):::userAction --> B[录入长宽与外壳刚性约束限制]:::userAction
    end

    subgraph Phase2 [② AI Agent 对话式智能 BOM 配置]
        B --> C{通过右侧边栏输入自然语言}:::userAction
        C -->|例如：再进线处加俩断路器| D[LangGraph 意图剥离与组件查询]:::aiAction
        D --> E[侧边栏打字机反馈与画布动态刷新]:::sysAction
        E --> F{清单检查是否齐备?}:::userAction
        F -->|查漏补缺| C
    end

    subgraph Phase3 [③ 基于 RAG 的一键智能落盘排版]
        F -->|物料确立| G(用户发起：使用历史经验排版):::userAction
        G --> H[系统提取部件高维结构特征]:::sysAction
        H --> I[检索并推选 TopN 近似历史图档]:::aiAction
        I --> J(对比组件增减人工点击采纳):::userAction
        J --> K[运筹学防干扰算法启动全图对焦就位]:::sysAction
    end

    subgraph Phase4 [④ 交互式布局微操与输出交付]
        K --> L(在可视区选取不满意器件进行强拽):::userAction
        L --> M[触发自动吸附与自动排挤退让算法]:::sysAction
        M --> N(排布满足装配需求进行终审确认):::userAction
        N --> O([一键打包输出报表及装配数据包]):::userAction
    end
```

## 2. 交互场景 1：AI 工具箱对话配置元器件

此流程详述了用户在右侧栏通过自然对话，完成高门槛元器件清单配置的业务动作。

```mermaid
sequenceDiagram
    participant User as 🔌 电气工程师
    participant UI as 🖥️ 平台操作视图
    participant AI as 🤖 智能需求助手

    User->>UI: 在侧边对话框下达自然指令 (例如："进线柜增加两个电流互感器")
    UI->>AI: 传递用户意图
    
    activate AI
    AI-->>UI: 💡 界面开始打字机式流式展现 AI 意图理解过程
    AI->>AI: 匹配标准件库与电气常识规则
    AI-->>UI: ⚡ 立即下发局部更新指令 (无需等待语言说完)
    UI->>UI: 画布上的设备组件树瞬间刷出一堆新零件图标
    AI-->>UI: 💬 生成结束确认文本："好的，互感器已为您加上。"
    deactivate AI
    
    User->>UI: 肉眼评估树状结构确认无漏洞
```

## 3. 交互场景 2：基于沉淀图档的智能排版

此页面描述一旦清单“配齐”，系统如何调用沉淀图纸库帮用户完成初始布局。

```mermaid
sequenceDiagram
    participant User as 🔌 电气工程师
    participant UI as 🖥️ 平台排版视图
    participant RAG as 🧠 模板参考中心
    participant Core as ⚙️ 碰撞避让引擎

    User->>UI: 点击全局功能按钮“进行 AI 排版”
    UI->>RAG: 根据当下的部件向云端发起检索请求
    
    activate RAG
    RAG->>RAG: 计算特征并从历史优秀图档库寻找近义模板
    RAG-->>UI: 响应返回最精准的前 3 个历史布局方案
    deactivate RAG
    
    UI-->>User: 开启卡片浮窗，向用户明确展示：该推荐模板对比当前“多出什么零件”、“少了哪些零件”
    User->>UI: 点击采纳最优的一套
    
    UI->>Core: 将用户方案及模板方案联合送入算法空间
    activate Core
    Core->>Core: 让老零件对应、新零件补位 
    Core->>Core: 刚性运算消灭所有的重叠冲突干涉现象
    Core-->>UI: 输出绝对精确的画布座标分布结果
    deactivate Core
    
    UI->>UI: 前端动画一闪，上百个堆在一起的元器件瞬间按规矩排列整齐落位
    User->>UI: 检查大致布局满意，进入最后微调操作
```

## 4. 交互场景 3：排版画布上的微操感体验

本流程展现当系统排出版面后，由于现场接线习惯或业主私人要求，人工需要介入的畅快操作体验。

```mermaid
sequenceDiagram
    participant User as 🔌 电气工程师
    participant Canvas as 🎨 交互二维画布
    participant Logic as 📏 画布体验交互逻辑

    User->>Canvas: 鼠标左键点选目标“断路器”并开始强行拖动
    Canvas->>Logic: 提供实时坐标移动轨迹响应
    
    activate Logic
    Logic->>Logic: 检测元器件外缘与其他组件及边框的距离
    Logic-->>Canvas: 🧲 【吸附】即将靠齐的瞬间提供自动磁吸咬合，对齐更省眼力
    
    Logic->>Logic: 如果不慎拖到一组已密排完毕的接触器上重叠
    Logic-->>Canvas: 🌊 【自动推流】受撞击组件向周围合理退让，挤出对应空间给目标进场
    deactivate Logic
    
    User->>Canvas: 视觉达到最优态，松开左键完成物理落子
    Canvas->>Canvas: 保存布局微调的增量修改
```
