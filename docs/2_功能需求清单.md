# Layout-RAG 智能辅助设计平台：系统功能需求清单

本清单明确了系统各个模块的具体功能定义。每一项功能均通过后端的 FastAPI、LangGraph 引擎及前台 Vue 响应式状态完成闭环，具备极高的工程可溯源性。

## 1. 核心业务与状态管理领域
| 功能点名称 | 功能详细约束与验收标准 | 底层代码与架构映射 | 优先级 |
| :--- | :--- | :--- | :--- |
| **标准领域定义 (Domain)** | 系统能基于不同工业场景（如低压柜）约束业务边界。 | 继承自 `BusinessDomain` 基类（如 `LowvoltageCabinetDomain`），定义特定的特征 `schema` 与预处理逻辑。 | P0 |
| **方案生命周期管理** | 支持用户在浏览器端创建、切换、保存完整的电气柜方案 JSON 对象。 | 前端 Vue3 维护的 `schema` 响应式对象树；支持多面板 (`panels`) 以及嵌套部件级对象的序列化。 | P0 |

## 2. AI 智能配置引擎 (Agentic Configuration)
| 功能点名称 | 功能详细约束与验收标准 | 底层代码与架构映射 | 优先级 |
| :--- | :--- | :--- | :--- |
| **Token级流式对话交互** | 系统内置聊天侧边栏，用户指令与大模型推理必须具备打字机流式（Streaming）体验并实时响应。 | 基于 `agent_endpoints.py` 中的 `astream_events` (v2)，通过 SSE 接口推送 `{"type": "token", ...}` 帧。 | P0 |
| **智能思考链路展示** | 在回答复杂计算或逻辑时，支持将大模型的内部结构化推理链路展示给用户。 | 通过 SSE 拦截 `reasoning_content`，下发 `{"type": "thinking"}` 帧进行特殊 UI 渲染。 | P1 |
| **工具路由与状态突变** | 智能体不只是回话，须具备改变当前项目方案（增删改元件）的能力。 | `LangGraph` 状态机执行 Tool Calling，输出 `{"type": "action"}` 并直接驱动 Vue 前端方案重绘。 | P0 |
| **多模态图面解析** | 允许用户上传铭牌、面板草图，系统自动将其转换为规范化的零部件清单 JSON。 | 在 `HumanMessage` 中混合 `image_url`，请求支持 Vision 能力的大模型（如 GPT-4o）。 | P1 |

## 3. RAG 布局算法检索与自动解算
| 功能点名称 | 功能详细约束与验收标准 | 底层代码与架构映射 | 优先级 |
| :--- | :--- | :--- | :--- |
| **加权特征向量召回** | 基于当前设计的元器件清单，检索出 `templates` 中最匹配的 N 个历史布局模板。 | `VectorStore.search` 使用 Min-Max 及 Log1p 后计算加权欧氏距离完成比对。 | P0 |
| **候选模板 Diff 报告** | 对每个推荐候选，必须明确且量化地向用户告知“缺失元件”与“超额元件”项。 | 在推荐过程中自动生成并计算 `featureDiffs`，用以影响推荐 Score。 | P0 |
| **四阶级联坐标迁移** | 对齐历史模板坐标时，必须遵循“同类主选 -> 备用补位 -> 游标顺延 -> 兜底铺设”的先后顺序。 | 对应 `LayoutOptimizer` 中的 `_match_parts_to_template` 和 `_resolve_unmatched_targets` 方法。 | P0 |
| **CP-SAT 刚性空间解算** | 所有推荐坐标必须在执行时消除任何形式的边界交叉存留与物理干涉。 | 调用 `Google OR-Tools` 的 `AddNoOverlap2D`，求出加权 L1 偏差最小的 `arrange` 输出结果。 | P0 |

## 4. 交互式视图与可视化排版
| 功能点名称 | 功能详细约束与验收标准 | 底层代码与架构映射 | 优先级 |
| :--- | :--- | :--- | :--- |
| **友好布局交互编辑器** | 提供可视化的画布和交互式操作器，支持元器件拖拽、微调，确保所见即所得的可视化排版体验。 | 前端视窗交互，通过事件修饰符拦截并计算偏移量，实时更新 Vue 维护的 `schema` 响应状态树。 | P0 |
| **智能吸附与自动挤出** | 拖拽操作中，元器件靠近其他元器件边缘、导轨或网格时提供磁性自动吸附（Snapping）；同时支持拖拽遇到碰撞时的自动挤出（Collision & Push）排版特性。 | 在前端交互层实施 AABB 碰撞检测机制与推算法，确保流畅的微操编辑体验。 | P1 |
