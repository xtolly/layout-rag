# Layout-RAG 智能辅助设计平台：需求背景与痛点分析

## 1. 业务现状与架构愿景
在低压柜及配电集成领域，元器件（如微型断路器、接触器、保护装置等）的物理布局与二维走线是设计的核心。传统工作流高度依赖工程师在 CAD 或 Eplan 等专业软件中手动选型并划定元器件座标。由于不同项目间存在大量的拓扑相似性，未能将历史盘柜排版经验进行工程化、数字化沉淀，导致了极大的效率浪费。

为解决上述问题，本项目引入了 **Layout-RAG** 架构。系统并不试图通过端到端的黑盒大模型生成坐标（难以保证 100% 的物理准确性），而是采用了 **“自然语言智能体驱动录入 -> 基于特征的加权向量召回 (RAG) -> 数学约束规划求解 (CP-Sat)”** 的白盒化、高确定性技术路径。

## 2. 工程痛点对标与技术应对方案

### 痛点一：参数配置门槛高，人工录入易错（前端提效）
**业务现状**：
电气柜型号（`cabinet_model`）、用途及元器件选型涉及大量复杂的行业知识。工程师在配置清单时，需要翻阅手册核对尺寸（`part_size`）与参数，表单录入极易因“错填”、“漏填”导致后续物理空间不足。
**技术应对机制**：
- 引入基于 `LangGraph` 状态机编排的 **Configurator Agent（配置智能体）**。
- 支持用户通过自然语言指令（如“新增一台 1600A 进线柜并配备两个微断”）或上传图纸/铭牌照片（利用模型的 Vision 视觉能力），触发代理底层的 `Tool Calling`，直接将非结构化意图转换为标准的内部配置 JSON (`schema`)。

### 痛点二：历史优秀图纸复用率极低（数据资产唤醒）
**业务现状**：
企业沉淀了数以万计的成功柜体设计图。但由于这些历史方案以非结构化图形文件存在，工程师面对新需求时，难以从海量历史中找出“长宽相近、器件类似”的参考模板，最终往往重头设计。
**技术应对机制**：
- 设计专用的 **特征抽提引擎 (`FeatureExtractor`)**，从 JSON 树的 `meta.parts` 节点提取连续特征（面积统计）与动态分类特征（`part_type`）。
- 离线计算生成本地 `NumPy (.npz)` 压缩特征矩阵，实现在线的亚秒级相似模板加权召回（基于加权欧氏距离的相似度评估）。

### 痛点三：布局碰撞难以杜绝，缺乏硬性兜底（规则硬约束）
**业务现状**：
新手工程师参照相似项目修改时，常常因元器件尺寸的变更或边距把控不当，导致元件在图纸上产生重叠干涉，这种错误往往要到车间装配调试时才暴露，造成昂贵的返工成本。
**技术应对机制**：
- 摒弃了缺乏物理边界约束的传统遗传算法或纯 LLM 推理。
- 采用 **Google OR-Tools (CP-SAT 约束规划求解器)**，在系统 `LayoutOptimizer` 阶段，建立 `AddNoOverlap2D` 硬约束与面板安全边距约束，通过数学规划求解出最小偏移坐标，保证 100% 物理无干涉。

### 痛点四：纯算法排版缺乏微调手段，人工干预繁琐（交互体验优化）
**业务现状**：
即使有算法辅助，部分细节布局仍需要人工进行最后微调以满足特定项目习惯。而在传统工具中微调时，手工对齐和干涉处理费时费力。
**技术应对机制**：
- 前端实现一套**友好的布局交互编辑器**，为工程师提供即开即用的“所见即所得”视图。
- 深度集成**自动吸附（Snapping）**对齐辅线以及**自动挤出（组件推流避让）**的高级交互反馈，让人工干预阶段的操作更加丝滑高效。

## 3. 总体建设目标
本系统致力于通过严谨的软件工程抽象，实现低压配电柜设计中“配置流”与“排版流”的双向智能辅助。这并非一个玩具原型的 POC，而是一个面向产线交付、基于领域驱动设计（DDD）、能做到参数强校验与零物理干涉的工业级智能排版专家平台。
